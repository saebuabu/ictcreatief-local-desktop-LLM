# MBO Experimenten met Ollama + Open WebUI

Een verzameling praktische experimenten voor studenten en docenten op het MBO,
georganiseerd per afdeling. Alle experimenten draaien lokaal — geen internetverbinding nodig,
geen privacyproblemen met leerlingdata.

---

## Algemene tips voor docenten

- Stel een **Modelfile** in met een Nederlandse systeem-prompt voor betere resultaten
- Gebruik `ollama run` in de terminal als introductie, Open WebUI voor dagelijks gebruik
- Laat studenten modellen **vergelijken** (llama3, mistral, gemma) — dat leert kritisch denken
- Benadruk altijd: AI heeft het **niet altijd goed**, controleer altijd de output

## Software Development (Applicatieontwikkeling)

### Voor studenten

**Experiment 1: Code Review Assistant**
- Plak eigen code in de chat
- Vraag: *"Wat zijn de verbeterpunten in deze code?"*
- Doel: leren feedback ontvangen en verwerken

**Experiment 2: Bug Finder**
- Schrijf bewust code met een bug erin
- Laat de AI de bug vinden en uitleggen
- Doel: debuggen leren + AI als hulpmiddel begrijpen

**Experiment 3: Documentatie schrijven**
- Geef een functie/klasse aan de AI
- Vraag om docstrings, commentaar, README
- Doel: belang van documentatie inzien

**Experiment 4: Code uitleggen**
- Geef onbekende code (bijv. van GitHub)
- Vraag: *"Leg deze code uit alsof ik een beginner ben"*
- Doel: code leren lezen

**Experiment 5: Taal-naar-taal vertaling**
- Schrijf een functie in Python
- Vraag om conversie naar JavaScript of C#
- Doel: taalstructuren vergelijken

**Experiment 6: Unit tests genereren**
- Geef een functie aan de AI
- Vraag om unit tests te schrijven
- Doel: testgedreven ontwikkeling leren

### Voor docenten

- Laat AI een **opdracht genereren** op basis van leerdoelen
- Gebruik AI om **meerdere niveaus** van dezelfde opdracht te maken (niveau 2, 3, 4)
- Laat studenten de AI-output **beoordelen** als peer-review oefening

## Klassikale experimenten (alle afdelingen)

### De AI Vergelijkingstest
- Stel dezelfde vraag aan 3 verschillende modellen (llama3, mistral, gemma)
- Vergelijk de antwoorden: wat verschilt, wat klopt, wat niet?

### De Hallucination Hunt
- Laat AI iets vertellen over een (fictief) onderwerp
- Controleer de feiten via andere bronnen
- Doel: kritisch denken en bronnenonderzoek

### Prompt Engineering Workshop
- Stel dezelfde vraag op 5 verschillende manieren
- Vergelijk de resultaten
- Doel: begrijpen hoe je AI effectief aanstuurt

### Privacy & Ethiek Discussie
- Wat mag je wel/niet aan een AI vertellen?
- Wanneer is AI nuttig, wanneer gevaarlijk?
- Welke beroepen veranderen door AI?

## Modelfile voor MBO gebruik

Maak een eigen model aan dat beter past bij MBO-onderwijs:

```
FROM llama3.2

SYSTEM """
Je bent een behulpzame assistent voor MBO-studenten en docenten in Nederland.
Je communiceert in duidelijk Nederlands, op B1/B2-taalniveau.
Je legt begrippen stap voor stap uit.
Je geeft altijd aan wanneer je ergens niet zeker over bent.
Je moedigt studenten aan om informatie te controleren via andere bronnen.
"""

PARAMETER temperature 0.7
PARAMETER num_ctx 4096
```

Sla op als `Modelfile` en maak het model aan:
```bash
ollama create mbo-assistent -f Modelfile
ollama run mbo-assistent
```

## Open WebUI starten

Open WebUI biedt een ChatGPT-achtige interface voor Ollama, geschikt voor dagelijks gebruik door studenten en docenten.

**Vereisten:**
- Docker Desktop geïnstalleerd en actief
- Ollama draait op de achtergrond

**Starten:**
```bash
docker compose up -d
```

Open daarna **http://localhost:3000** in de browser.

**Stoppen:**
```bash
docker compose down
```

> Het `docker-compose.yml` bestand staat in de root van dit project.
> Alle chatgeschiedenis en instellingen blijven bewaard in een Docker volume.
